# 잠재적 오류 점검 메모

이 문서는 현재 코드 기준으로 사용 중 발생할 수 있는 잠재적 오류와 개선 후보를 정리한 메모입니다. 실제 장애가 확정된 항목이 아니라, 재현 조건과 영향도를 확인해 볼 필요가 있는 위험 목록입니다.

## 요약

| 우선순위 | 영역 | 잠재적 오류 | 영향 |
| --- | --- | --- | --- |
| 높음 | ICNS 변환 | `convert_done_body` 문구가 항상 ICO 생성으로 표시됨 | ICNS 변환 성공 후 사용자에게 잘못된 안내 표시 |
| 높음 | 언어 변경 | `subprocess.Popen([sys.executable, *sys.argv])`가 패키징 환경에서 새 프로세스를 올바르게 실행하지 못할 수 있음 | 언어 변경 시 앱 재시작 실패 가능 |
| 중간 | 초기 파일 경로 | 환경변수로 전달된 초기 파일을 검증하지 않고 표시함 | 삭제되었거나 지원하지 않는 파일도 선택된 것처럼 보일 수 있음 |
| 중간 | ICNS 크기 | 1024px 계열 ICNS 리소스를 생성하지 않음 | 일부 macOS 환경에서 고해상도 아이콘 품질 저하 가능 |
| 중간 | 출력 상태 | 원본 선택 직후 출력 경로가 `.ico`로 고정 표시됨 | ICNS 변환을 누르기 전까지 예상 출력 경로가 헷갈릴 수 있음 |
| 낮음 | 오류 메시지 | 변환 함수의 확장자 오류 메시지가 영어로 고정됨 | 한국어 UI에서도 일부 오류가 영어로 노출 |

## 상세 항목

### 1. ICNS 변환 성공 문구가 ICO로 표시됨

- 위치: `i18n.py`의 `convert_done_body` 문자열
- 현재 증상: `Convert to ICNS` 버튼으로 변환해도 성공 팝업은 `ICO file created.` / `ICO 파일을 생성했습니다.`라고 안내합니다.
- 예상 영향: 결과 파일은 `.icns`로 생성되더라도 사용자가 실제 생성 포맷을 혼동할 수 있습니다.
- 개선 방향:
  - 포맷별 성공 문구를 분리합니다. 예: `ico_convert_done_body`, `icns_convert_done_body`
  - 또는 `convert_done_body`에 `{format}` 플레이스홀더를 추가하고 호출부에서 `format_name`을 전달합니다.

### 2. 패키징 환경에서 언어 변경 재시작이 실패할 수 있음

- 위치: `main.py`의 `_restart_with_language()`
- 현재 구현: `subprocess.Popen([sys.executable, *sys.argv], env=env)`로 현재 실행 명령을 다시 실행합니다.
- 잠재 위험:
  - PyInstaller one-file 또는 macOS `.app` 환경에서는 `sys.executable`과 `sys.argv` 조합이 개발 환경과 다르게 동작할 수 있습니다.
  - `sys.argv`에 상대 경로나 추가 인자가 포함된 경우 재시작 대상이 의도와 달라질 수 있습니다.
- 예상 영향: 언어 변경 직후 기존 창은 닫히지만 새 창이 뜨지 않을 수 있습니다.
- 개선 방향:
  - 패키징 모드 여부(`getattr(sys, "frozen", False)`)에 따라 실행 명령을 분기합니다.
  - 재시작 실패 시 기존 창을 닫지 않고 오류를 보여주는 방어 로직을 추가합니다.

### 3. 초기 원본 파일 경로 검증 부족

- 위치: `IconBundlerApp.__init__()`와 `_sync_source_display()`
- 현재 구현: `ICON_BUNDLER_SOURCE` 환경변수 값이 있으면 곧바로 `Path(initial_source)`로 저장합니다.
- 잠재 위험:
  - 언어 변경 사이에 원본 파일이 삭제되거나 이동될 수 있습니다.
  - 지원하지 않는 확장자의 경로가 환경변수로 주입될 수 있습니다.
- 예상 영향: UI에는 파일이 선택된 것으로 표시되지만 변환 시점에 오류가 발생합니다.
- 개선 방향:
  - 초기화 시 `exists()`, `is_file()`, 지원 확장자 여부를 확인합니다.
  - 검증 실패 시 `source_path`를 `None`으로 되돌리고 상태 메시지로 안내합니다.

### 4. ICNS 고해상도 리소스 누락 가능성

- 위치: `ICNS_SIZES`
- 현재 크기 목록: `16`, `32`, `64`, `128`, `256`, `512`
- 잠재 위험: macOS 아이콘은 Retina 표시를 위해 1024px급 리소스가 사용되는 경우가 있습니다.
- 예상 영향: 큰 화면이나 Finder 미리보기에서 아이콘이 덜 선명하게 보일 수 있습니다.
- 개선 방향:
  - `ICNS_SIZES`에 `(1024, 1024)`를 추가하는 방안을 검토합니다.
  - 1024px보다 작은 원본의 업스케일 정책을 명확히 정합니다.

### 5. 원본 선택 직후 출력 경로가 ICO로 고정 표시됨

- 위치: `select_source_file()`와 `_sync_source_display()`
- 현재 증상: 원본 파일을 선택하면 출력 경로가 항상 `.ico`로 표시됩니다.
- 잠재 위험: 사용자가 `Convert to ICNS`를 누를 예정이어도 UI상 예상 출력이 `.ico`로 보여 혼동할 수 있습니다.
- 개선 방향:
  - 출력 경로 표시를 버튼 클릭 이후 갱신하거나, 두 출력 후보를 모두 표시합니다.
  - 버튼별로 예상 출력 경로를 별도 안내합니다.

### 6. 일부 오류 메시지가 고정 영어 문구로 노출됨

- 위치: `convert_image()`의 확장자 검증 오류
- 현재 구현: `ValueError("PNG, JPG, JPEG files only.")`
- 잠재 위험: 한국어 UI 상태에서도 변환 함수에서 직접 발생한 오류는 영어로 표시됩니다.
- 개선 방향:
  - 예외 메시지는 내부 코드로 유지하고 UI 계층에서 `get_text()`로 변환합니다.
  - 또는 `convert_image()`가 구체적인 예외 타입을 발생시키고 호출부에서 현지화 문구를 선택합니다.
